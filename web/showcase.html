01

diskwhy scan

Quick scan of your home directory. Finds space hogs by category and shows a size-sorted list with staleness scores. The disk header shows total capacity and how much is in use.

zsh — ~
~ diskwhy scan Scanning /Users/dj (quick) ... macOS / Macintosh HD · 499.9 GB total · 312.3 GB used · 187.6 GB free FOUND 4.2 GB across 8 items ───────────────────────────────────────────────────────────────── 1 node_modules ~/old-app/node_modules 1.8 GB unused 2 docker 3 dangling images · 2 unused volumes 923 MB — 3 xcode_derived ~/Library/Developer/Xcode/DerivedData631 MB stale 4 node_modules ~/blog/node_modules 412 MB recent 5 pycache ~/scripts/__pycache__ 248 MB stale 6 brew_cache ~/Library/Caches/Homebrew 178 MB unused 7 git_objects ~/monorepo/.git 94 MB stale 8 logs ~/Library/Logs 81 MB stale ───────────────────────────────────────────────────────────────── Run: diskwhy clean --all --dry-run ~ ▋
Flag Description
--path <dir> Scan a specific directory instead of home. Shows scan_mode: "path" in JSON output.
--deep Also walks /usr/local, /opt/homebrew, /var/cache, and other system directories. Shows scan_mode: "deep".
--json Emit structured JSON with schema_version: 1. Safe to pipe to jq or parse in scripts.
--verbose Show per-directory timing, resolved symlink paths, and staleness source (atime / mtime / sentinel).
02

diskwhy clean

Always run --dry-run first to preview what will be deleted. Active items are unconditionally skipped. When you're ready, remove --dry-run and confirm.

preview with --dry-run
diskwhy clean --all --dry-run
~ diskwhy clean --all --dry-run Scanning (deep) ... CLEAN PREVIEW — no changes will be made ───────────────────────────────────────────────────── ✓ dry-run node_modules ~/old-app/node_modules 1.8 GB ✓ dry-run xcode_derived ~/Library/Developer/Xcode631 MB ✓ dry-run pycache ~/scripts/__pycache__ 248 MB ✓ dry-run brew_cache ~/Library/Caches/Homebrew178 MB ✓ dry-run git_objects ~/monorepo/.git 94 MB → gc ✓ dry-run logs ~/Library/Logs 81 MB ✗ skipped node_modules ~/blog/node_modules 412 MB active ───────────────────────────────────────────────────── Recoverable: 3.0 GB | 1 item skipped (active) Docker: 3 images + 2 volumes = 923 MB (included) Run without --dry-run to apply. ~ ▋
run it
diskwhy clean --all --yes
~ diskwhy clean --all --yes Scanning (deep) ... CLEANING 6 items · 3.0 GB recoverable ───────────────────────────────────────────────────── ✓ deleted node_modules ~/old-app/node_modules 1.8 GB ✓ deleted xcode_derived ~/Library/Developer/Xcode631 MB ✓ deleted pycache ~/scripts/__pycache__ 248 MB ✓ deleted brew_cache ~/Library/Caches/Homebrew178 MB ✓ gc run git_objects ~/monorepo/.git 94 MB ✓ deleted logs ~/Library/Logs 81 MB ✗ skipped node_modules ~/blog/node_modules 412 MB active ───────────────────────────────────────────────────── Docker pruned: 3 images + 2 volumes 923 MB ───────────────────────────────────────────────────── Freed: 3.9 GB in 6 operations (1 skipped) ~ ▋

Every item in the output has an outcome. The six possible states:

dry_run would have been deleted
deleted removed from disk
gc_run git gc ran on the repo
trashed moved to system Trash
skipped active, filtered, or blocklisted
error deletion failed (permissions, etc.)
03

diskwhy shell

Run diskwhy with no arguments to open the interactive REPL. Slash commands for scan, clean, help, and more. Useful when you want to run multiple operations without retyping the binary. Adapts to dark and light terminals.

zsh — diskwhy shell

But why? Made by DJ Commands: /scan [--deep] [--path <dir>] /clean [--all|--node|--cache|--git|--logs] [--dry-run] [--trash] [--yes] /version /help /home /clear /exit diskwhy> /scan --path ~/projects/webapp Scanning ~/projects/webapp ... FOUND 2.1 GB across 3 items ───────────────────────────────────────────── 1 node_modules node_modules 1.9 GB stale 2 pycache __pycache__ 89 MB stale 3 git_objects .git 94 MB stale diskwhy> /clean --node --cache --yes Scanning ... ✓ deleted node_modules 1.9 GB ✓ deleted pycache 89 MB Freed: 1.99 GB diskwhy> /exit bye ~ ▋
Command Description
/scan [flags]Run a scan. Accepts the same flags as diskwhy scan.
/clean [flags]Run clean. Accepts the same flags as diskwhy clean.
/helpShow available commands and examples.
/homeRe-render the branded home screen.
/versionPrint the binary version.
/clearClear the terminal screen.
/exitQuit the shell.

JSON output

Both diskwhy scan and diskwhy clean support --json. The output has a stable schema_version: 1 contract. Pipe to jq, parse in CI, or feed into dashboards.

diskwhy scan --json
scan_output.json
schema_version: 1
{ "schema_version": 1, "scanned_at": "2026-01-15T10:30:42Z", "scan_mode": "quick", "header": "[macOS / Macintosh HD]", "disk": { "total_bytes": 499963174912, "used_bytes": 312345678901, "free_bytes": 187617496011, "mount": "/" }, "items": [ { "path": "/Users/dj/old-app/node_modules", "category": "node_modules", "size_bytes": 1879048192, "staleness_score": "unused", "staleness_source": "atime" }, // ... more items ], "docker": { "dangling_images": 3, "dangling_bytes": 967884800, "unused_volumes": 2, "unused_vol_bytes": 8388608, "containers_total": 4, "containers_stopped": 1 }, "summary": { "total_items": 8, "total_bytes": 4509715456, "elapsed_ms": 834 } }

pipe to jq

05

Shell completion

Tab completion for all flags and subcommands, generated by cobra's built-in completion engine. Install once and forget.

bash
diskwhy completion bash > /usr/local/etc/bash_completion.d/diskwhy
source ~/.bashrc
zsh
diskwhy completion zsh > "${fpath[1]}/_diskwhy"
source ~/.zshrc
fish
diskwhy completion fish > ~/.config/fish/completions/diskwhy.fish
powershell
diskwhy completion powershell | Out-String | Invoke-Expression
